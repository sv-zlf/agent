## grep

Searches file contents using regex patterns - the fastest way to find code or text within files.

**Usage:**
- Searching for specific code patterns
- Finding text within files
- Regex-based content search

**Parameters:**
- `pattern` (required): Regex pattern to search for
- `path` (optional): Root directory to search (defaults to current directory)
- `include` (optional): File pattern to filter (e.g., "*.ts")

**Regex Examples:**
- `function getUserById` - Find function definitions
- `import.*from.*react` - Find React imports
- `TODO|FIXME` - Find todo comments
- `class.*extends.*Component` - Find class definitions
- `async.*function` - Find async functions

**Example:**
```json
{
  "tool": "grep",
  "parameters": {
    "pattern": "function.*test",
    "path": "H:/Project/agent/src",
    "include": "*.ts"
  }
}
```

**Best Practices:**
- Use Grep for searching file contents with regex
- Prefer Grep over Bash's grep command
- Combine with Glob for efficient exploration
- Use simple, specific patterns for better results
