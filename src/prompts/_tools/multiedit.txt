## multiedit

Performs multiple edit operations on a single file atomically.

**When to use:**
- Multiple edits to the SAME file
- All edits must succeed or all fail (atomic)

**When NOT to use:**
- Editing different files → use separate `edit` calls
- Single edit → use `edit`

**Parameters:**
- `filePath` (required): Absolute path to the file
- `edits` (required): Array of edit objects, each with `oldString` and `newString`

**Example:**
```json
{
  "tool": "multiedit",
  "parameters": {
    "filePath": "H:/Project/agent/src/config.ts",
    "edits": [
      {
        "oldString": "const API_URL = 'old';",
        "newString": "const API_URL = 'new';"
      },
      {
        "oldString": "const TIMEOUT = 5000;",
        "newString": "const TIMEOUT = 10000;"
      }
    ]
  }
}
```

**Important Notes:**
- All edits are applied sequentially
- If any edit fails, ALL changes are rolled back
- Maximum 50 edit operations per call
