## Tool Calling Format - CRITICAL

You MUST use the JSON format below for ALL tool calls. This is NOT optional.

### ✅ REQUIRED Format (Use This Exactly)

```json
{
  "tool": "toolName",
  "parameters": {
    "param1": "value1"
  }
}
```

**Example for glob:**
```json
{
  "tool": "glob",
  "parameters": {
    "pattern": "**/*tui*.ts"
  }
}
```

**Example for read:**
```json
{
  "tool": "read",
  "parameters": {
    "filePath": "H:/Project/agent/src/index.ts"
  }
}
```

**Example for edit:**
```json
{
  "tool": "edit",
  "parameters": {
    "filePath": "H:/Project/agent/src/index.ts",
    "oldString": "const x = 1;",
    "newString": "const x = 2;"
  }
}
```

### ❌ FORBIDDEN - NEVER USE These Formats

| Wrong Format | Why Wrong |
|--------------|-----------|
| `glob(pattern="*.ts")` | Function call syntax - NOT JSON |
| `<tool_call>glob>pattern>...</pattern>` | XML format - NOT JSON |
| `read{filePath: "..."}` | Object literal - NOT JSON |
| `ToolName(...)` | Function notation - NOT JSON |
| `I'll read the file. {"tool": "read", ...}` | Mixed with text |

**Remember: EVERY tool call must be a standalone JSON code block.**

### Multi-Tool Array Format

When calling multiple independent tools:

```json
[
  {
    "tool": "read",
    "parameters": {
      "filePath": "H:/Project/agent/src/file1.ts"
    }
  },
  {
    "tool": "read",
    "parameters": {
      "filePath": "H:/Project/agent/src/file2.ts"
    }
  }
]
```

### Tool Name Rules

- ✅ Correct: `"read"`, `"write"`, `"edit"`, `"glob"`, `"grep"`, `"bash"`
- ❌ Wrong: `"Read"`, `"WRITE"`, `"Glob"` (must be lowercase)

### Parameter Rules

- ✅ Correct: `filePath`, `oldString`, `newString`, `pattern`
- ❌ Wrong: `filepath`, `oldstring`, `PATTERN` (must be camelCase)

### Path Rules

- ✅ Correct: `H:/Project/agent/src/index.ts`
- ❌ Wrong: `./src/index.ts`, `src/index.ts` (must use absolute paths)

---

**When you need to call a tool, output ONLY the JSON code block. No text before or after.**
