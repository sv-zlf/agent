Searches file contents using regex patterns - the fastest way to find code or text within files.

**Usage:**
- The pattern parameter specifies the regex pattern to search for
- The path parameter specifies the root directory (defaults to current directory)
- Searches are case-insensitive
- Returns matching lines with file paths and line numbers

**Format:**
```json
{
  "tool": "grep",
  "parameters": {
    "pattern": "function.*test",
    "path": "H:/Project/agent/src"
  }
}
```

**Regex Examples:**
- `function getUserById` - Find function definitions
- `import.*from.*react` - Find React imports
- `TODO|FIXME` - Find todo comments
- `class.*extends.*Component` - Find class definitions
- `async.*function` - Find async function declarations

**Recommendations:**
- Use Grep for searching file contents with regex patterns
- Prefer Grep over Bash's grep command for better performance
- Combine with Glob tool for efficient codebase exploration
- Use simple, specific patterns for better results
- Test regex patterns incrementally if not getting expected results

**pattern** parameter is required - the regex pattern to search for
