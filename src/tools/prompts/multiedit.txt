Performs multiple edit operations on a single file. All edits are executed sequentially as atomic operations.

**When to use multiedit:**
- When you need to edit the SAME file multiple times → use `multiedit`
- When editing DIFFERENT files → use separate `edit` calls

**REQUIRED Format:**
```json
{
  "tool": "multiedit",
  "parameters": {
    "filePath": "H:/Project/agent/src/utils/config.ts",
    "edits": [
      {
        "oldString": "export const API_URL = 'old'",
        "newString": "export const API_URL = 'new'"
      },
      {
        "oldString": "export const TIMEOUT = 5000",
        "newString": "export const TIMEOUT = 10000"
      }
    ]
  }
}
```

**Important Notes:**
- All edits are applied sequentially
- If any edit fails, ALL changes are rolled back
- Use `oldString` (NOT `oldtext`) and `newString` (NOT `newtext`)
- Maximum 50 edit operations per call
